# 说明
使用flyway进行数据库的结构和数据自动升级

可以把升级脚本放到该目录，并以 V1__create_user_ddl.sql,V2__add_user_dml.sql 格式

# 两种文件
## V开头 
仅需要被执行一次的SQL命名以大写的"V"开头，后面跟上"0~9"数字的组合,数字之间可以用“.”或者下划线"_"分割开，然后再以两个下划线分割，其后跟文件名称，最后以.sql结尾。比如，V2.1.5__create_user_ddl.sql、V4.1_2__add_user_dml.sql。

## R开头
可重复运行的SQL，则以大写的“R”开头，后面再以两个下划线分割，其后跟文件名称，最后以.sql结尾。。比如，R__truncate_user_dml.sql。

## 注意：
- 其中，V开头的SQL执行优先级要比R开头的SQL优先级高。
- 如果V开头的执行过了就不能在修改，应新增文件

- 不要随意修改已经提交并且可能被用户执行过的 Flyway 脚本（如 V1__xxx.sql），否则会导致校验失败（Flyway 会认为脚本被篡改）。
- 如果你发现已发布的脚本有错误，通常应该通过新增一个修正脚本（如 V2__fix_v1_error.sql）来解决，而不是修改旧的 V1脚本。
